# Privorum Modernization Design Brief

## Direction

Privorum should adopt a visual style that is:

- 75% Quiet Enterprise
- 25% Editorial Tech

This means the website should feel:

- premium
- calm
- technically credible
- modern
- structured
- distinctive without being loud

The site should not feel like:

- a crypto hype landing page
- a generic AI SaaS template
- a dark hacker dashboard
- a playful chatbot brand

## Brand Position

Recommended position:

**Privorum helps companies design and ship AI workflows, backend platforms, and workflow automation systems, with blockchain and distributed-systems expertise when needed.**

This should shape both the design and the copy.

## Visual Principles

### 1. Trust First

The visual system should immediately suggest:

- reliability
- engineering maturity
- senior technical judgment

### 2. Controlled Personality

The site should have more character than a standard consulting website, but that character should come from:

- typography
- spacing
- composition
- tone

Not from visual noise.

### 3. Systems, Not Hype

Visual language should suggest:

- orchestration
- structure
- flow
- precision

Avoid obvious crypto and AI cliches.

## Palette

Recommended palette direction:

- `Ink`: `#17191d`
- `Graphite`: `#2a2f38`
- `Steel`: `#657181`
- `Mist`: `#d9dee5`
- `Cloud`: `#eef1f4`
- `Paper`: `#f7f5f0`
- `Cobalt`: `#245dff`
- `Soft Cobalt`: `#5f86ff`
- `Mineral`: `#5f7463`

### Usage

- `Paper` for the primary background
- `Cloud` and `Mist` for section contrast and surfaces
- `Ink` and `Graphite` for text and dark surfaces
- `Cobalt` for links, primary CTA, and key highlights
- `Steel` for secondary text and UI details
- `Mineral` only as a minor accent if needed

### Color Notes

- Keep the site mostly neutral
- Use blue with discipline
- Do not overuse secondary accents
- Avoid coral as a dominant supporting color in the redesign

## Typography

Recommended direction:

- Headings: elegant editorial serif or refined display serif
- Body and UI: neutral modern sans-serif

### Suggested Pairings

#### Option A

- Headings: `Fraunces`
- Body/UI: `Manrope`

#### Option B

- Headings: `Cormorant Garamond`
- Body/UI: `IBM Plex Sans`

#### Option C

- Headings: `Baskervville`
- Body/UI: `Plus Jakarta Sans`

### Recommendation

Use **Option A** if the site should feel more modern.

Use **Option B** if the site should lean more technical and slightly more sober.

Best fit for Privorum:

**Headings: Fraunces**
**Body/UI: IBM Plex Sans or Manrope**

## Layout System

### Grid

- use a clean central content column
- allow selected sections to break the grid slightly for editorial character
- keep client proof and service sections aligned and structured

### Spacing

- increase whitespace substantially compared with the current theme
- larger top and bottom section spacing
- more generous gaps between proof items and content blocks

### Surfaces

- alternate between `Paper` and `Cloud`
- use occasional dark `Ink` sections for contrast if needed
- avoid too many bordered boxes

## Visual Motifs

Use subtle motifs that imply systems and orchestration:

- soft grid lines
- thin dividers
- node-like alignment patterns
- layered planes
- abstract flow shapes
- minimal data or systems textures

Do not use:

- coins
- chains
- glowing neon nodes
- chat bubble overload
- generic AI illustration art

## Homepage Wireframe

## 1. Hero

Goal:

- explain what Privorum does now
- establish trust quickly
- signal technical depth

Structure:

- short eyebrow
- strong headline
- supporting paragraph
- primary CTA
- secondary CTA
- quiet system-inspired visual

Suggested content shape:

- Eyebrow: `AI Systems, Backend Platforms, Distributed Software`
- Headline: clear AI-first statement
- Supporting text: explain AI workflows, automation, orchestration, and production systems

## 2. Capability Pillars

Four blocks:

- AI Workflows
- Workflow Automation
- Backend Platforms
- Blockchain & Distributed Systems

These should be cleaner and more specific than the current feature blocks.

## 3. Client Proof

Goal:

- show that Privorum has depth across older and newer work
- support the message without making the page feel crowded

Structure:

- strong section title
- compact introduction
- clean grid of logos or names
- optional short line of supporting text

## 4. What Privorum Helps With

Examples:

- connect LLMs to real systems
- automate internal or customer-facing workflows
- build production backend services
- integrate with complex platforms and data sources
- ship reliable systems with observability and scale in mind

## 5. Services Preview

A tighter version of the services list with stronger wording and less generic copy.

## 6. Why Privorum

A trust section emphasizing:

- senior technical leadership
- hands-on delivery
- production systems experience
- AI and blockchain depth

## 7. Final CTA

Tone:

- direct
- calm
- premium

Examples:

- `Discuss Your System`
- `Plan an AI Workflow`
- `Book a Technical Consultation`

## Clients Page Styling Direction

The client list should stay unified.

Design treatment:

- clean, structured grid
- equal spacing
- neutral card or panel treatment
- optional hover emphasis
- no noisy shadows
- short, edited descriptions

The page should feel like a proof archive, not a marketing collage.

## Services Page Styling Direction

The service pages should become:

- more outcome-oriented
- easier to scan
- less text-heavy
- more technically specific

Recommended structure:

- service headline
- short positioning statement
- key outcomes
- technologies / systems involved
- delivery model
- CTA

## Component Style Guide

### Buttons

- solid cobalt primary button
- neutral secondary button
- crisp corners or slightly rounded corners
- premium hover states, not playful animation

### Cards

- restrained borders
- soft surface contrast
- strong typography
- generous internal spacing

### Navigation

- cleaner top bar
- more spacing
- stronger active state
- less theme-default styling

### Footer

- darker and more refined than the current version
- fewer but stronger elements
- include contact and trust cues

## Motion

Use motion carefully:

- subtle fade and rise on load
- staggered entrance for proof and service blocks
- restrained hover transitions
- no excessive parallax or playful micro-interactions

## Implementation Phases

### Phase 1

- update palette variables
- update typography
- redesign homepage hero and feature section
- improve buttons, navigation, and footer

### Phase 2

- redesign services list and service detail pages
- modernize clients page styling
- improve CTA patterns

### Phase 3

- refine supporting pages
- replace weaker legacy illustrations
- tighten mobile layouts and rhythm

## Success Criteria

The redesign is successful if the site feels:

- more premium
- more modern
- more credible
- more aligned with AI-first positioning
- less like a reused Hugo theme

And if a visitor can understand within seconds that Privorum:

- builds serious technical systems
- works on AI and automation
- still has deep blockchain and distributed-systems experience
