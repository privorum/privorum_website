# Privorum Site Update Plan

## Purpose

This document summarizes the current website update direction for Privorum.

The immediate goals are:

- make the site clearer and more credible
- keep all client proof in one place
- prepare the brand for a stronger AI workflow, automation, and backend systems focus

## Clients Page Decision

The site should use **one unified Clients page**.

Do not split the page into:

- clients
- ventures
- leadership experience

For this website update, all of that proof should appear together under a single `Clients` section.

## What Has Been Updated

The clients page has been simplified back into one list.

The page now:

- keeps a single `Clients` heading
- includes older and newer client names together
- removes the split between ventures and selected experience

## Positioning Direction

The site should still move toward a newer market position.

Recommended direction:

**Privorum helps companies design and ship AI workflows, workflow automation, and production-grade backend platforms.**

This should be the new story on the homepage and services pages, while the clients page remains the proof layer and blockchain remains supporting depth where relevant.
